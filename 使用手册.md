## 六大功能详解

### 1. /brief - 每日简报

**作用**：获取今日市场分析和持仓建议

**触发方式**：
- "简报"
- "今日市场"
- "持仓分析"
- `/brief`

**输出内容**：
- 宏观形势（政策、市场、外围）
- 持仓分析（风险预警、操作建议）
- 明日关注要点

**使用场景**：
- 每天开盘前/收盘后查看
- 了解当前持仓状态

---

### 2. /scan - 市场扫描

**作用**：发现投资机会，推荐标的

**触发方式**：
- "有什么机会"
- "推荐"
- "扫描市场"
- `/scan`
- `/scan AI` - 聚焦某个方向

**输出内容**：
- 市场热点和资金动向
- 3个推荐标的（含买点、止损位）
- 关注池更新建议

**使用场景**：
- 想找新的投资标的
- 周末做下周规划

---

### 3. /analyze - 个股分析

**作用**：深度分析单个股票或ETF

**触发方式**：
- "分析下腾讯"
- "比亚迪怎么样"
- "159995值得买吗"
- `/analyze 00700`

**输出内容**：
- 公司/ETF概况
- 财务分析
- 估值分析
- 风险评估
- 操作建议

**使用场景**：
- 买入前做功课
- 定期检视持仓

---

### 4. /trade - 交易记录

**作用**：记录买卖操作，更新持仓

**触发方式**：
- "今天买了xxx"
- "卖出了xxx"
- "加仓了xxx"
- "今天没操作"
- `/trade`

**执行动作**：
- 验证价格真实性
- 更新 `Holdings.md`
- 记录到 `trades.md`
- 设置验证时点

**使用场景**：
- 每次交易后记录
- 保持数据准确

---

### 5. /review - 周期复盘

**作用**：复盘交易决策，验证准确性

**触发方式**：
- "复盘"
- "回顾"
- "这周怎么样"
- `/review`
- `/review week` - 周复盘
- `/review month` - 月复盘

**输出内容**：
- 交易回顾（对了还是错了）
- 建议执行情况
- 经验总结
- 准则更新建议

**使用场景**：
- 每周末进行
- 每月末全面复盘

---

### 6. /committee - 投资委员会

**作用**：多模型协同分析，提取共识

**触发方式**：
- "开会"
- "投资委员会"
- "多模型分析"
- `/committee`
- `/committee 红利ETF该怎么操作`

**执行流程**：
1. Claude 先给出分析
2. 你去 Cursor/Copilot 获取 Codex 意见
3. 你去 Gemini 获取 Gemini 意见
4. 回来让 Claude 汇总共识

**输出内容**：
- 强共识（3/3一致）
- 弱共识（2/3一致）
- 分歧区（需你决策）

**使用场景**：
- 重大决策前
- 不确定该怎么操作时

## 技能同步（Claude ↔ Codex）

**目录位置**：
- Claude skills：`.claude/skills/`
- Codex skills：`~/.codex/skills/`

**一键同步脚本**：`./sync_skills.sh`

```
# 默认：Claude -> Codex
./sync_skills.sh

# 反向同步：Codex -> Claude
./sync_skills.sh --to-claude

# 严格镜像：删除目标中不存在的技能（保留 .system）
./sync_skills.sh --mirror
```

**注意**：
- Skills 有新增/修改/删除时，两边都要同步，避免行为不一致
- 同步后建议重启 Codex CLI（或新开会话）以加载新技能

## 数据来源说明

### 优先级

```
1. AKShare脚本 (最可靠)
   └── 价格、涨跌、盈亏、宏观数据、北向资金、快讯

2. 配置文件 (你维护)
   └── 持仓、关注池、投资风格

3. WebSearch (谨慎使用)
   └── 仅用于公司公告、政策原文、研报
```

### 数据更新

- **实时数据**：每次运行 skill 时自动获取
- **持仓数据**：`/trade` 后自动更新
- **分析报告**：`/analyze` 后自动保存

## 多模型协同操作指南

### 准备工作

确保你有以下工具：
- **Claude Code** (当前)
- **Cursor + Copilot** 或 **ChatGPT**
- **Gemini** (网页版或 API)

### 操作步骤

```
第一步：在 Claude Code 中说"开会"
        ↓
第二步：Claude 生成分析并保存
        ↓
第三步：打开 Cursor，粘贴相同输入数据
        让 Copilot/Codex 分析，保存结果
        ↓
第四步：打开 Gemini，同样操作
        ↓
第五步：回到 Claude Code，说"三个都弄好了"
        ↓
第六步：Claude 汇总共识，生成决策建议
```

### 模板文件

- 提示词模板：`Committee/Templates/prompt_template.md`
- 观点模板：`Committee/Templates/opinion_template.md`
- 共识模板：`Committee/Templates/consensus_template.md`

## 常见问题

### Q: 数据获取失败怎么办？

A: 脚本可能因网络问题失败，稍后重试。**不要使用猜测的价格**。

### Q: 持仓数据不对怎么办？

A: 手动编辑 `股市信息/Config/Holdings.md`，这是唯一真相源。

### Q: 简报中的持仓和实际不符？

A: 之前是硬编码问题，现已修复。如果仍有问题，检查 Holdings.md。

### Q: WebSearch 的信息准确吗？

A: 可能有时效误差。**价格数据永远以脚本输出为准**，搜索结果仅作参考。

### Q: 多模型意见不一致怎么办？

A: 这正是多模型的价值。看分歧原因，结合你自己的判断做决策。

## 最佳实践

### 日常流程

```
早盘前：/brief 看简报
盘中：有交易就 /trade 记录
收盘后：如有需要 /scan 找机会
周末：/review 复盘
```

### 重大决策

```
1. 先 /analyze 深度分析标的
2. 再 /committee 多模型讨论
3. 最后自己做决定
4. /trade 记录决策理由
```

### 定期维护

```
每周：检查 Watchlist.md 更新关注池
每月：/review month 全面复盘
每季：更新 Profile.md 和 Principles.md
```
